## 4. コアシステム設計

### 4.1 マップ生成（BSP法: Binary Space Partitioning）

**アルゴリズム概要:**
1. マップ全体を1つの矩形領域とする（例: 50x40タイル）
2. 矩形を再帰的に縦 or 横に分割（4〜6回）
3. 各リーフノードに「部屋」を配置（最小 4x4、最大 10x10）
4. 隣接する部屋同士を「通路」で接続
5. 部屋内にランダムで敵・アイテム・階段を配置

**タイルの種類:**

| タイルID | 名前 | 表示 | 通行 | 説明 |
|----------|------|------|------|------|
| 0 | 壁 | `#` (灰色) | 不可 | 通り抜け不可 |
| 1 | 床 | `.` (茶色) | 可能 | 通常の移動可能タイル |
| 2 | 通路 | `.` (暗茶) | 可能 | 部屋を接続する通路 |
| 3 | 階段(下) | `>` (黄色) | 可能 | 次の階層へ進む |
| 4 | ドア | `+` (茶色) | 可能 | 部屋の入口（将来拡張用） |

**マップデータ構造:**
```typescript
type TileType = 'wall' | 'floor' | 'corridor' | 'stairs' | 'door';

interface DungeonMap {
  width: number;       // マップの横幅(タイル数)
  height: number;      // マップの縦幅(タイル数)
  tiles: TileType[][]; // 2次元配列 [y][x]
  rooms: Room[];       // 生成された部屋のリスト
  explored: boolean[][]; // 探索済みフラグ（ミニマップ用）
}

interface Room {
  x: number;      // 左上X座標
  y: number;      // 左上Y座標
  width: number;
  height: number;
  center: { x: number; y: number }; // 部屋の中心座標
}
```

**基底クラス:**

```typescript
interface Entity {
```

### 4.3 敵キャラクター

**敵の種類（フロアに応じて出現）:**

| 名前 | 表示 | HP | ATK | DEF | AI | 出現階 | 経験値 |
|------|------|-----|-----|-----|-----|-------|--------|
| スライム | `s` (緑) | 15 | 4 | 1 | ランダム移動 | B1〜 | 10 |
| ゴブリン | `g` (赤) | 25 | 7 | 2 | 追跡型 | B2〜 | 25 |
| スケルトン | `S` (白) | 35 | 10 | 4 | 追跡型 | B3〜 | 40 |
| ダークメイジ | `M` (紫) | 20 | 15 | 2 | 遠距離攻撃 | B4〜 | 55 |
| ドラゴン | `D` (赤) | 80 | 20 | 8 | 追跡+範囲攻撃 | B5(ボス) | 200 |

**敵AIの行動パターン:**

1. **ランダム移動型**: 視界外ではランダムに移動。視界内に入ったら追跡開始
2. **追跡型**: プレイヤーを発見したらA*経路探索で追跡。隣接時に攻撃
3. **遠距離攻撃型**: プレイヤーとの距離が3以内なら遠距離攻撃、近づかれたら逃げる

### 4.4 戦闘システム（ターン制）

**ダメージ計算式:**
```
ダメージ = max(1, 攻撃側ATK - 防御側DEF + random(-2, 2))
```

**ターン進行:**
1. プレイヤーが方向キーを押す
2. 移動先に敵がいれば → 攻撃（移動しない）
3. 移動先が床/通路なら → 移動
4. 移動先が壁なら → 何も起こらない（ターン消費しない）
5. プレイヤーのターン終了 → 全ての敵が1回ずつ行動
6. 画面を再描画

**レベルアップ:**
```
必要経験値 = レベル × 50
レベルアップ時:
  - maxHP += 15
  - ATK += 2
  - DEF += 1
  - HP全回復
```

### 4.5 アイテムシステム

**アイテムの種類:**

| カテゴリ | 名前 | 表示 | 効果 | 出現率 |
|----------|------|------|------|--------|
| 回復 | 回復ポーション | `!` (赤) | HP+30回復 | 高 |
| 回復 | 大回復ポーション | `!` (ピンク) | HP+60回復 | 低 |
| 武器 | 鉄の剣 | `/` (灰) | ATK+3 | 中 |
| 武器 | 炎の剣 | `/` (赤) | ATK+7 | 低 |
| 防具 | 革の鎧 | `[` (茶) | DEF+2 | 中 |
| 防具 | 鋼の鎧 | `[` (灰) | DEF+5 | 低 |
| 特殊 | スクロール | `?` (青) | 周囲の敵に10ダメージ | 低 |

**アイテム操作:**
- マップ上のアイテムに重なると自動拾得 → インベントリに追加
- `I` キーでインベントリ表示
- 数字キーでアイテム使用/装備
- インベントリ上限: 10個

### 4.6 視界システム（FOV: Field of View）

**レイキャスティング方式:**
- プレイヤーの位置から360度方向にレイを飛ばす
- 壁に当たったらそこで停止
- レイが通った床タイルを「可視」にする
- 一度可視になったタイルは「探索済み」として暗く表示（ミニマップにも反映）

**視界半径:** 6タイル（デフォルト）

### 4.7 スコアシステム

```
スコア = (倒した敵の経験値合計) + (到達フロア × 100) + (残りHP × 2)
```

## 6. 技術詳細

### 6.1 開発環境

| 項目 | 選定 | 理由 |
|------|------|------|
| 言語 | TypeScript | 型安全・補完が効きやすい |
| バンドラー | Vite | 高速HMR・設定最小限 |
| テスト | Vitest | Viteと統合・高速 |
| フォーマッター | Prettier | コード整形 |
| リンター | ESLint | コード品質 |

### 6.2 package.json scripts

```json
{
  "scripts": {
    "dev": "vite",
    "build": "tsc && vite build",
    "preview": "vite preview",
    "test": "vitest"
  }
}
```

### 6.3 描画方式

- HTML5 Canvas 2D Context を使用
- タイルベース描画（テキスト文字を `fillText` で描画）
- 1ターンごとに全画面を再描画（ターン制のため60fps不要）
- 将来的にスプライト画像への差し替えが可能な構造にする

### 6.4 ゲーム状態管理

```typescript
type GameState = 'title' | 'playing' | 'inventory' | 'gameover';

interface GameContext {
  state: GameState;
  player: Player;

  floor: number;         // 現在の階層
  messageLog: string[];  // メッセージログ（最新20件保持）
  turnCount: number;     // ターン数
}
```

## 8. キー操作一覧

| キー | 動作 | 状態 |
|------|------|------|
| `←` / `A` | 左に移動（敵がいれば攻撃） | playing |
| `→` / `D` | 右に移動（敵がいれば攻撃） | playing |
| `↑` / `W` | 上に移動（敵がいれば攻撃） | playing |
| `↓` / `S` | 下に移動（敵がいれば攻撃） | playing |
| `I` | インベントリを開く/閉じる | playing |
| `1`〜`9` | インベントリ内のアイテムを使用 | inventory |
| `>` | 階段を降りる（階段の上にいるとき） | playing |
| `.` / `Space` | 待機（1ターン経過） | playing |
| `Enter` | 決定 | title / gameover |
| `Escape` | キャンセル / 閉じる | inventory |

## 11. 将来的な拡張案（ライブデモ向け即興ネタ）

- **宝箱**: 部屋にランダム配置、開けるとアイテム入手
- **罠**: 踏むとダメージ or デバフ（毒、鈍足）
- **NPC**: 回復してくれるNPCや、ヒントをくれるNPC
- **ショップ**: ゴールドで買い物
- **マップギミック**: テレポート床、一方通行ドア
- **マルチエンディング**: ボス撃破方法で結末が変化
- **AI実況**: ゲーム内イベントをAIが実況コメント
